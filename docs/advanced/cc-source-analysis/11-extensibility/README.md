# 11 — 扩展机制

## 核心问题
Claude Code 的 4 种扩展机制（Hook/Skill/Plugin/MCP）如何协作？成本对比是什么？

## 关键源码
- `src/skills/` — 技能系统
- `src/plugins/` — 插件系统
- `src/services/mcp/` — MCP 协议
- `src/utils/hooks.ts` (5022 行) — Hook 系统
- `src/schemas/hooks.ts` — Hook 配置

## 主要内容

### 1. 4 种扩展机制对比
| 机制 | 成本 | 模型参与 | 适用场景 |
|------|------|---------|---------|
| Hook | 零成本 | 不参与 | 确定性操作（事件触发） |
| Skill | 低成本 | 按需调用 | 知识复用（动作模板） |
| Plugin | 中成本 | 独立上下文 | 打包工具集 |
| MCP | 高成本 | 完整集成 | 外部服务（工具桥） |

### 2. Hook 系统
```typescript
// 事件触发的确定性脚本
// 模型完全不参与
// 零成本，最快
// 适用：代码格式化、lint 检查、日志记录
```

### 3. Skill 系统
```typescript
// 带元数据的 workflow package
// 可复用的动作模板
// 模型按需调用
// 适用：API 调用、发布流程、代码审查
```

### 4. Plugin 系统
```typescript
// 打包的工具集
// 有独立上下文
// 可以包含多个工具
// 适用：第三方集成
```

### 5. MCP（Model Context Protocol）
```typescript
// 完整的外部服务集成
// 工具桥 + 行为说明注入
// 一个 MCP 服务的工具描述可能占几千 token
// 接五六个 MCP 光工具列表就吃掉十分之一上下文
```

### 6. 成本优化原则
```
能用 Hook 解决的不用 Skill
能用 Skill 解决的不上 Plugin
能用 Plugin 解决的不上 MCP
越轻量越好
```

### 7. 生态的关键：模型"感知到"自己的能力
- 工具描述要清晰
- 模型要知道自己有什么能力
- 避免工具冲突

## 学习目标
- 理解 4 种扩展机制的差异
- 掌握成本优化原则
- 明白生态设计的关键
