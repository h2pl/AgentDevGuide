# 01 — 整体架构

## 核心问题
Claude Code 的 51 万行代码是如何组织的？为什么 AI 决策逻辑只占 1.6%，其余 98.4% 是什么？

## 关键源码
- `src/main.tsx` (4683 行) — 主入口
- `src/query.ts` (1729 行) — Agent 循环
- `src/QueryEngine.ts` (1295 行) — 查询引擎
- `src/Tool.ts` (792 行) — 工具基类
- `src/tools/` — 40+ 工具实现

## 主要内容

### 1. 五层架构全景
- CLI 入口层：Commander.js + React/Ink
- Agent 循环层：while(true) 状态机
- 工具执行层：40+ 工具的注册与调度
- 权限控制层：多层安全检查
- 系统提示层：53KB 动态提示词构建

### 2. 代码规模分布
| 模块 | 文件数 | 代码行数 | 占比 |
|------|--------|----------|------|
| 终端 UI | 389 | ~70,000 | 14% |
| 工具系统 | 184 | ~29,000 | 6% |
| 命令系统 | 207 | ~15,000 | 3% |
| 权限安全 | - | ~60,000 | 12% |
| Agent 核心 | - | ~8,000 | 1.6% |
| 基础设施 | 564 | ~330,000 | 64% |

### 3. Kernel + Harness 架构
- 极薄的 AI 内核（~88 行 while 循环）
- 极厚的工程外壳（权限、压缩、缓存、持久化）
- 设计哲学：模型负责思考，Harness 负责兜底

### 4. 模块依赖关系
- 入口层 → 引擎层 → 工具层 → 服务层
- 统一注册中心：tools.ts、commands.ts
- 功能门控：feature() 编译期 + GrowthBook 运行期

## 学习目标
- 理解 Claude Code 不是"API 包装器"，而是完整的 Agent 操作系统
- 掌握五层架构的职责划分
- 明白为什么 98.4% 的代码是工程基础设施
