# 核心机制：表示、对齐与融合

## 三大机制全景

如果把多模态模型比作一个人，那这三个机制就是它的核心认知能力：

```
表示（Representation） → 对齐（Alignment） → 融合（Fusion）
 "怎么看见"              "怎么关联"            "怎么综合理解"
```

- **表示**：把不同格式的输入（像素、声波、文字）变成模型内部统一的"语言"（向量）
- **对齐**：学会"这个向量（图）和那个向量（文字）说的是同一件事"
- **融合**：把多个模态的向量综合在一起，形成跨模态的理解

三个机制环环相扣。表示不当 → 对齐失败 → 融合混乱。让我们逐个拆开。

## 表示：把世界变成向量

神经网络只能理解数字，而且是高维向量。表示机制就是把不同模态的原始数据转换成向量。

### 文本：Token + Embedding

文本的表示是最成熟的。流程：

```
"一只猫在睡觉"
    ↓ Token 分词器
["一只", "猫", "在", "睡觉"]
    ↓ Embedding 层（可训练的查表映射）
[0.23, -0.15, 0.78, ...]  →  每个 Token 变成一个 4096 维的向量
[0.41,  0.33, -0.12, ...]
[-0.05, 0.67,  0.22, ...]
[0.89, -0.44, 0.01, ...]
```

这些向量不是随机数——Embedding 层通过大规模预训练学会了把语义相近的词映射到向量空间中相近的位置。"猫"和"狗"的向量距离，远小于"猫"和"汽车"的向量距离。

### 图像：ViT Patch 编码

图像表示的核心是 ViT（Vision Transformer）。它的思路很简单：

```
1024×1024 的图片
    ↓ 切成 16×16 的小方块（Patch）
64×64 = 4096 个 Patch
    ↓ 每个 Patch 展开成向量 + 位置编码
4096 个向量，每个 768 维
    ↓ 进入 Transformer 层
输出 4096 个图像特征向量
```

**关键理解**：ViT 把一张连续像素组成的图片，变成了一组离散的、和文本 Token 结构相同的向量。这为后续的多模态对齐创造了条件——图像向量和文本向量在数据结构上统一了。

### 音频：频谱 + Transformer

```
原始音频波形（1 秒 16000 个采样点）
    ↓ 短时傅里叶变换（STFT）
梅尔频谱图（128 个频带 × 100 个时间帧）
    ↓ 切成 Patch + 位置编码
多个频谱向量
    ↓ Transformer
音频特征向量
```

Whisper 和 GPT-4o 的语音处理都基于这个思路——把连续声波变成二维频谱图，然后用类似视觉模型的 Patch 方式处理。

### 原生统一的 Token 化

第三代模型（GPT-4o/Gemini）的突破在于**不再需要分别编码**。图片直接按照像素区域 Token 化，和文本 Token 一起输入同一个 Transformer。这意味着：

- 图片的"理解"不需要预先经过一个视觉模型——Transformer 在自注意力层中直接建立像素 Token 和文字 Token 的关联
- 没有信息损失——不需要 ViT 先把图片"压缩"为少量特征向量
- 但计算量巨大——一张高清图片会变成数千甚至上万个 Token

## 对齐：让图像和文字互相理解

表示让不同模态都变成了向量。但问题来了：一个代表"猫的图片"的向量，和一个代表文字"猫"的向量，在数学上有关系吗？**默认没有。** 对齐机制就是建立这种关系。

### 方案一：对比学习（CLIP 的方式）

CLIP 是 2021 年 OpenAI 提出的对齐方案，至今仍是多模态领域的基石。

**训练方式**：准备 4 亿对（图片, 文字描述）数据。同时训练一个图像编码器和一个文本编码器。对于每一对正确的图文，让它们的向量尽可能接近；对于不匹配的，让它们推远。

```
正确的图文对：
"一只橘猫" ←→ 🐱图片  → 向量距离小 ✓

错误的图文对：
"一只橘猫" ←→ 🐕图片  → 向量距离大 ✓
```

训练完成后，CLIP 学会了把语义相同的图片和文字映射到向量空间中**同一个位置附近**。这是所有后续多模态模型的基础——LLaVA、BLIP-2、Qwen-VL 都直接用了 CLIP 的视觉编码器。

### 方案二：Q-Former（BLIP-2 的方式）

CLIP 的对齐是"让整个图片和整段文字对齐"，粒度太粗。一张图片里有多个物体，Q-Former 解决的是**精细化对齐**。

Q-Former 内部有一组可学习的"查询向量"（Query Tokens），它们的作用相当于"带着问题去看图"：

- "图片里有什么物体？"→ 查询向量去提取物体相关特征
- "这些物体之间的关系是什么？"→ 查询向量去提取关系特征
- "整体场景描述是什么？"→ 查询向量去提取全局特征

然后 Q-Former 只把和文本问题最相关的视觉特征传给 LLM，过滤掉无关信息。这是一种"按需提取"的对齐方式。

### 方案三：简单投影（LLaVA 的方式）

LLaVA 走了一条更极简的路：用一个简单的线性变换层（MLP），直接把 CLIP 提取的图像特征投影到 LLM 的 Token 嵌入空间。

惊人的是，**这个简单方案的效果出奇的好**。它证明了一个关键事实：CLIP 已经做了足够好的图文对齐，LLM 本身有足够强的语义理解能力，中间只需要一个最简单的"格式转换"就够了。

### 方案四：原生交叉注意力（GPT-4o/Gemini 的方式）

第三代模型不再需要单独的对齐模块——图片 Token 和文本 Token 直接在同一层 Transformer 里做自注意力。

Transformer 的自注意力机制天然适合跨模态对齐——模型可以自动学习哪些图片 Token 和哪些文字 Token 应该关联。这种对齐是可微的、端到端的、不受预设方式限制的。

## 融合：把多模态信息合在一起

有了对齐后的多模态向量，下一步是**融合**——综合所有模态的信息做出判断。

### 三种融合策略

| 策略 | 方式 | 优点 | 缺点 | 典型应用 |
|------|------|------|------|---------|
| **早期融合** | 所有模态的原始 Token 一起输入模型 | 信息最完整 | 计算量大 | GPT-4o 原生统一 |
| **中期融合** | 各模态分别编码到中间层，在 Transformer 内交汇 | 平衡效率和信息 | 架构设计复杂 | LLaVA、Qwen-VL |
| **后期融合** | 各模态独立推理，在决策层合并结果 | 最简单 | 缺跨模态交互 | 传统多模型集成 |

绝大多数现代多模态模型使用**中期融合**——在 Transformer 的各个层里，文本 Token 和图像 Token 通过自注意力机制持续交互。每个 Token 在每一层都会"看"其他所有 Token（包括不同模态的），从而逐步加深跨模态理解。

### 注意力融合的原理

Transformer 的自注意力机制在融合中起到了核心作用：

```
输入：文本Token [T1, T2, T3] + 图像Token [I1, I2, I3, I4]

自注意力计算：
T1 "关注" → T1, T2, T3, I1, I2, I3, I4  （文字关注了所有图片区域）
I1 "关注" → T1, T2, T3, I1, I2, I3, I4  （图片区域关注了所有文字和图片区域）
```

经过多层自注意力后，每个 Token 都"知道"了其他所有模态的信息。模型在最后一层综合所有 Token 的表示，做出最终判断。

## 实战视角：这些机制怎么影响你选模型

| 如果你想... | 选哪个模型 | 机制层面的原因 |
|------------|-----------|---------------|
| 低成本本地部署 | LLaVA/Qwen-VL | 使用 CLIP 对齐 + 简单投影，计算量小 |
| 文档OCR+图表分析 | Qwen-VL | 高分辨率输入 + 中文专项对齐训练 |
| 复杂多模态推理 | GPT-4o/Claude | 原生统一架构，无信息损失，跨模态注意力最强 |
| 视频理解 | Gemini | 原生多模态训练包含视频数据，时序注意力 |
| 自定义微调 | Qwen-VL（开源）| 对齐和融合模块清晰，方便针对业务场景微调 |

## 总结

- 多模态模型的三大核心机制：**表示**（编码为向量）→ **对齐**（建立跨模态关联）→ **融合**（综合推理）
- **表示**的核心是 ViT Patch 编码——把图片切成小块，和文本 Token 统一为相同数据结构
- **对齐**有四种方案：CLIP 对比学习（粗粒度全局对齐）、Q-Former（精细化局部对齐）、简单投影（依赖 CLIP + LLM 能力）、原生交叉注意力（端到端，不预设）
- **融合**的引擎是自注意力机制——不同模态的 Token 在 Transformer 的每一层互相"看"，逐步建立深层跨模态理解
