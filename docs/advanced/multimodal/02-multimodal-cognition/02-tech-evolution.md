> 多模态模型的发展有一条清晰的三代演进路线：从"视觉模型拍张照、文字模型来读"到"一个模型同时看和读"。理解这条演进线，你才能看懂 01 章那些产品背后的技术更替。

你好，我是江小湖。上一篇讲了多模态 AI 的技术定义和基本挑战。这一篇沿着时间线走一遍——从 2021 年的 CLIP 到 2026 年的 GPT-4o，多模态模型的架构经历了三代根本性的变化。

## 演进总览：三代架构

```
2021-2022         2023-2024             2025-2026
第一代              第二代                 第三代
拼接式          →   指令微调式          →   原生统一
Pipeline             Instruction Tuning     Native Multimodal

视觉编码器+LLM      端到端微调             统一Token空间
CLIP ViT + GPT      LLaVA 1.5/1.6          GPT-4o
BLIP-2 + FlanT5     Qwen-VL                Gemini 2.5
MiniGPT-4           InstructBLIP           Claude 4
```

每一代解决的是上一代的致命短板。理解这个进化逻辑，比记住每个模型的名字更重要。

## 第一代：拼接式 Two-stage Pipeline

**思路简单直接：先让一个视觉模型看懂图，再把理解结果翻译成文字模型能读的格式。**

### 架构

```
图片 → [视觉编码器 ViT] → 图像特征 → [投影层 MLP] → LLM → 文字回答
```

三个组件各司其职：
- **视觉编码器**（通常用 CLIP 的 ViT）：把图片变成向量表示
- **投影层**（一个简单的 MLP 或多层感知机）：把视觉向量映射到 LLM 的 Token 空间
- **大语言模型**（LLaMA/FlanT5/Vicuna）：用映射后的视觉 Token + 用户文字 Prompt 综合推理，输出回答

### 代表模型

- **LLaVA**（2023，微软+威斯康星大学）：用 GPT-4 生成图片问答训练数据，首次跑通了"看图聊天"的体验
- **BLIP-2**（2023，Salesforce）：引入 Q-Former（Querying Transformer），用可学习的查询向量从视觉特征中提取与语言最相关的信息
- **MiniGPT-4**（2023，KAUST）：冻结视觉编码器和 LLM，只训练一个简单的线性投影层，证明了"桥接"本身就能做到不错的零样本效果

### 为什么它是革命性的

在拼接式之前，多模态理解主要靠"端到端训练一个专门的图文模型"——成本极高，而且无法利用 LLM 强大的推理和指令跟随能力。

拼接式的核心创新是 **"分而治之"**——复用现有最强的视觉模型和语言模型，只训练中间那个"翻译层"。训练成本大幅降低（LLaVA 只用 8 张 A100 训练一天），但效果惊人——从未见过的问题类型也能回答个大概。

### 致命短板

- **视觉信息损失**：ViT 提取的图像特征是压缩过的——一张照片经过编码后只剩几百个 Token，细节大量丢失。问模型"图里第三行第二个字是什么"，它看不清
- **错误传播**：视觉编码器理解错了 → 投影层传递了错误信息 → LLM 基于错误信息推理，整个链条不会自我纠正
- **不是真正的"理解"**：LLM 看到的是"一串代表图像特征的数字"，不是图像本身。它对图片的理解是二手信息

## 第二代：指令微调式 Instruction Tuning

**解决第一代的核心问题：让模型不再只是"读图片描述"，而是真正学会基于视觉信息回答问题。**

### 架构变化

```
图片 → [视觉编码器+投影层+LLM] → 端到端联合训练 → 文字回答
```

关键变化：
- **不再冻结 LLM**：在包含视觉问答、图表理解、OCR 的大规模数据集上对整个 Pipeline 进行微调
- **高质量指令数据**：不再是简单的"这张图里有什么"，而是"分析这张财报的营收趋势""比较两张设计稿的排版差异"，让模型学会深层视觉推理
- **大幅提升细节分辨率**：支持更高分辨率的输入图片，减少信息损失

### 代表模型

- **LLaVA 1.5**（2023.10）：在 LLaVA 基础上增加了学术任务导向的微调数据和高分辨率支持，效果大幅提升，成为开源标杆
- **LLaVA 1.6**（2024.01）：引入动态高分辨率策略——把大图切成多个子图分别编码再融合，解决了"小字看不清"的问题
- **Qwen-VL**（2023-2024，阿里巴巴）：在中文场景做到极致，OCR 和图表理解国内第一，支持多图对话和定位（在图上画框标注位置）
- **InstructBLIP**（2023，Salesforce）：对 BLIP-2 进行指令微调，首次证明 Q-Former 架构也可以通过微调大幅提升效果

### 核心突破

把拼接式模型从"能看图"变成了"能理解图"。第一代模型看到一个饼图可能只会说"这是一个饼图"，第二代模型能说出"营收占比最大的是电商业务，占 45%，同比增长 12 个百分点"。

### 遗留问题

- **仍是拼接架构**：视觉编码器和 LLM 还是两个独立训练的系统，靠投影层"翻译"。两张系统之间永远存在语义鸿沟
- **多模态不真正的统一**：处理文本和处理图像用的是两套参数、两套训练目标——模型内部没有真正"融合"两种模态的理解
- **不支持生成**：理解侧的 Transformer 和生成侧的扩散模型完全是两套系统

## 第三代：原生统一 Native Multimodal

**2025-2026 年的范式转变：不拼接任何东西，从头训练一个真正理解所有模态的统一模型。**

### 架构本质

```
文本 → [Token化]
图片 → [Token化]  →  统一的 Transformer  →  回答/生图/语音
音频 → [Token化]
```

核心思想：**把所有模态都变成 Token，喂给同一个模型。** 不再有"视觉编码器+投影层+LLM"的三件套，只有一个模型从头学到了如何在 Token 空间里统一表示文字、图片和声音。

### 关键技术创新

**统一 Token 化**。GPT-4o 直接把图片像素编码成 Token（不需要预先用 ViT 提取特征），和文本 Token 混在一起输入 Transformer。图片的"理解"不再经过二手转述——Transformer 自己在自注意力层中建立不同模态 Token 之间的关联。

**端到端训练**。不再分阶段——视觉编码器和 LLM 不是分别训练的。同一个目标函数、同一批参数、同时优化所有模态的理解能力。这让模型学会了真正的跨模态推理。

**原生生成能力**。因为所有模态在同一个 Token 空间，模型可以输出图片 Token 而不只文字 Token。GPT-4o 的原生生图（直接生成图片而非调用 DALL-E）就是建立在这个基础之上。

### 代表模型

- **GPT-4o**（2024.05，OpenAI）：首个大规模商用原生多模态模型，支持文本+图片+音频输入，文本+图片+音频输出
- **Gemini 2.5**（2025，Google）：从头设计的多模态架构，视频理解是核心优势，超长上下文窗口支持百万级 Token
- **Claude 4**（2025-2026，Anthropic）：在文档理解、UI 分析、Computer Use 场景的原生多模态优势突出

### 为什么这是终局方向

拼接式架构的局限在于"翻译不可避免会丢失信息"，指令微调只能在"更好的翻译"上努力。而原生统一架构消除了翻译这个环节——图片直接变成 Token，Transformer 直接看，没有中间商赚信息差。

**这意味着模型真的"看到"了你的图，而不是"读到了一份关于你图片的描述报告"。**

## 三代对比：一张表看清

| 维度 | 第一代 拼接式 | 第二代 指令微调 | 第三代 原生统一 |
|------|-------------|---------------|---------------|
| 代表模型 | LLaVA, BLIP-2 | LLaVA 1.5, Qwen-VL | GPT-4o, Gemini 2.5 |
| 架构 | ViT + 投影层 + LLM | ViT + 投影层 + 微调LLM | 统一 Token + 单一 Transformer |
| 视觉信息 | 二手（ViT 提取后转述） | 二手（但分辨率更高） | 一手（直接 Token 编码） |
| 训练方式 | 只训练投影层 | 联合微调 | 端到端统一训练 |
| 生成能力 | 不能生成 | 不能生成 | 原生支持多模态输出 |
| 跨模态推理 | 弱 | 中等 | 强 |
| 典型问题 | "图里小字看不清" | "复杂推理还不够深入" | "计算成本和延迟更高" |

## 为什么这条演进线很重要

这条演进线不是学术界的自娱自乐，它直接影响你作为开发者能做什么：

- **第一代模型（LLaVA/BLIP-2）**：本地部署成本极低，适合"有图就行"的简单场景（如物体识别、简单问答）。一张消费级显卡就能跑
- **第二代模型（LLaVA 1.5/Qwen-VL）**：适合需要深度理解的场景（文档分析、图表解读、OCR），中文场景 Qwen-VL 是首选
- **第三代模型（GPT-4o/Gemini/Claude）**：API 调用成本高，但能力最强。复杂推理、跨模态综合分析、生产级应用的唯一选择

**你不知道这条演进线，选模型就只能看价格和跑分。知道了，才能理解每个模型的能力边界从哪来。**

## 总结

- 多模态模型架构经历了**三代演进**：拼接式 → 指令微调 → 原生统一，每一代在解决上一代的根本缺陷
- **第一代的核心创新是"分而治之"**——复用最好的视觉模型和语言模型，代价是信息损失
- **第二代解决了"能看图→能理解图"的跳跃**——通过高质量指令微调让模型学会深层视觉推理
- **第三代取消"翻译"环节**——直接把所有模态变成 Token，端到端统一理解和生成
- 三代不是替代关系：第一代适合本地部署，第三代适合追求最高能力，第二代是当前开源路线的主流
