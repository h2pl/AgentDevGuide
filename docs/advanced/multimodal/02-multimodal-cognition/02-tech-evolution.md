## 第一代：拼接式 Two-stage Pipeline

### 核心思路

2023 年初，LLM（大语言模型）已经很强大了——它懂推理、能遵循复杂指令、能写代码。但它唯一的感知器官是文本——给它一张图，它无从下手。

拼接式架构解决这个问题的方式简单粗暴：**找一个"看图专家"（视觉编码器），让它把图片的内容写成一份"内部报告"，再交给"读书专家"（LLM）去理解和推理。**

### 架构细节

```
┌─────────┐      ┌──────────┐      ┌──────────┐
│  图片    │ ──→ │  视觉     │ ──→ │  投影层   │ ──→ LLM 可以读的向量
│  .jpg   │      │  编码器   │      │  (MLP)   │
└─────────┘      │  (ViT)   │      └──────────┘
                 └──────────┘           │
                                        ↓
                                    ┌──────────┐
                     用户文字 Prompt ──→ │   LLM   │ ──→ 文字回答
                                    │ (LLaMA/  │
                                    │ FlanT5)  │
                                    └──────────┘
```

三个组件各司其职：

**视觉编码器（通常是 CLIP ViT-L/14）**
- 输入：一张原始图片（如 336×336 像素）
- 处理：切成 24×24=576 个 Patch → 每个 Patch 变成 1024 维向量
- 输出：576 个图像特征向量
- 角色：把图像"翻译"成一种接近语言的形式

**投影层**
- 一个简单的线性变换（通常是单层 MLP 或一个可训练的矩阵 W）
- 功能：维度转换——把 1024 维的图像特征向量映射到 LLM 的 Token 嵌入空间（如 4096 维）
- 这是整个 Pipeline 里**唯一需要训练的组件**

**大语言模型（LLM）**
- 接收到：576 个映射后的图像 Token + 用户的文字 Token
- 做推理：这和读一段超长的文字没有区别——多模态输入被转化成了纯文本问题
- 输出：自然语言回答

### 关键模型及其创新

**LLaVA (Large Language and Vision Assistant)** — 2023 年 4 月，微软+威斯康星大学

LLaVA 是第一个让拼接式架构"能用"的模型。此前学术界有类似思路，但效果不佳。LLaVA 做了两件创造性的事：

**创新 1：GPT-4 生成训练数据**

通常训练一个图文问答模型需要人工标注大量"图片-问题-答案"三元组，成本极高。LLaVA 的做法：给 GPT-4（当时只支持文本输入）输入**图片的文字描述**（由 COCO 数据集提供），让它基于描述生成问答对。

```
输入给 GPT-4：
"图片描述：一个男人坐在长椅上，旁边放着自行车，背景是城市广场。
请基于这个场景生成 3 个问答对，包含各种类型的问题。"

GPT-4 输出：
Q1: 图中有几个人？ A1: 一个人。
Q2: 这个男人在做什么？ A2: 坐在长椅上。
Q3: 他的交通工具有什么特点？ A3: 旁边有一辆自行车，可能是骑过来的。
```

用 GPT-4 的回答作为 ground truth 去训练 LLaVA——**用最强的文本模型教视觉模型怎么"说人话"**。这个创意让 LLaVA 在只有 15.8 万条训练数据的情况下（对比其他模型的百万级），效果就达到了当时的 SOTA。

**创新 2：极简投影层**

LLaVA 的投影层就一个矩阵乘法：W × 图像特征。没有任何非线性激活、没有注意力机制、没有复杂的网络结构。

这在当时看来"太简单了，不可能好用"。但结果证明：**CLIP 已经做了足够好的图文对齐，LLaMA 已经有足够强的语义理解能力，中间只需要最轻量的格式转换就够了。**

```python
# LLaVA 投影层的核心逻辑（极大简化）
image_features = clip_vit.encode(image)      # shape: [576, 1024]
projection = nn.Linear(1024, 4096)            # 唯一可训练的参数
visual_tokens = projection(image_features)    # shape: [576, 4096]

# 拼接视觉Token和文本Token，送入LLM
input_tokens = torch.cat([visual_tokens, text_embeddings], dim=1)
output = llama_model.generate(input_tokens)
```

8 张 A100 GPU 训练不到一天，成本不到一千美元。

**BLIP-2** — 2023 年 1 月，Salesforce Research

BLIP-2 的贡献是一个更精妙的"翻译器"设计——**Q-Former（Querying Transformer）**。

Q-Former 是一组可学习的"查询向量"（32 个，每个 768 维），它们的工作方式是：

1. 第一步（自注意力）：查询向量之间互相交流，"我们要找什么？"
2. 第二步（交叉注意力）：查询向量"看"图像特征，"图片里哪些区域跟我们想问的有关？"
3. 第三步（自注意力）：查询向量再次交流，"我们各自看到了什么？"
4. 最终输出的 32 个向量，包含了从图片中提取的最相关视觉信息

对比 LLaVA 的 576 个向量全送 LLM，BLIP-2 只送 32 个精选向量——**效率更高，但丢失的信息也更多**。

**MiniGPT-4** — 2023 年 4 月，KAUST

做了一件事：冻结一个最强的视觉编码器（BLIP-2 的 EVA ViT-G）+ 冻结一个最强的 LLM（Vicuna-13B），只训练投影层。惊人的是，仅凭 Q-Former 的对齐能力 + LLM 的推理能力，零样本效果已经很好。

MiniGPT-4 的论文证明了一个事实：**拼接式架构的"翻译层"可以极简——因为翻译的两头（视觉编码器、LLM）已经各自足够强。**

### 为什么它是一次革命

拼接式之前，多模态模型需要从零训练——视觉和语言在一个模型里联合训练，成本是天文数字，而且模型能力被限制在那个固定组合里。

拼接式说："我不训练新模型，我复用现成的。" 视觉拿最好的（CLIP ViT），语言拿最好的（LLaMA/FlanT5），中间只训练一个翻译层。训练成本下降 100 倍以上，而效果提升了一个量级。

**这是多模态 AI 的"iPhone 时刻"——不是发明了新技术，而是把已有技术用正确的方式组合了起来。**

### 致命短板

如果有且只有一个最大的问题，那就是 **信息损失**。

LLM 看到的不是你的原始图片。它看到的是 ViT 提取的 576 个特征向量——每个向量 1024 个数。把这 576×1024=589,824 个数和原始图片的 3,145,728 个像素值对比——**信息压缩了约 5 倍**。

在大量信息丢失的情况下，这代模型经常犯错：
- 小字看不清：压缩后的特征向量丢掉了高分辨率的细节
- 精细物体分不清：同一张图里的"手表"和"手环"在特征层面可能非常接近
- 没有"回头看"的能力：LLM 只能根据那 576 个向量推理，不能实时要求更仔细地看某个区域

此外，**错误会传播**：ViT 看错了 → 投影层传错了 → LLM 基于错误信息推理，整个过程不会自我纠正。

## 第二代：指令微调式 Instruction Tuning

### 为什么需要第二代

拼接式模型会"看图说话"了，但说得很粗糙。你问它"第几季度营收最高"，它能回答"Q4"——因为柱状图最高的柱子特征明显。但你问它"Q3 到 Q4 的增长率是多少百分比"，它就卡住了——这需要把视觉信息（柱状图的高度）转化为精确的数值计算。

更本质地说：**拼接式模型是在"看图"和"读图里的文字"，不是在"理解图里的信息"。** 第二代要解决的就是从"能看到"到"能理解"的跳跃。

### 架构变化

```
拼接式：ViT(冻结) → 投影层(训练) → LLM(冻结)
                     ↑ 只训练这个

指令微调：ViT → 投影层 → LLM → 端到端微调
           ↑─────── ───────↑ 后两项联合优化
```

三个关键变化：

**变化 1：LLM 不再冻结**

拼接式训练中，LLM 的参数是固定的——它被告知"这里有 576 个向量代表一张图，请根据它们回答问题"。但这种"被告知"的效果有限——LLM 没有被训练成专门读取 ViT 编码的视觉向量。

微调阶段，LLM 的参数被优化，学会了**如何从 ViT 输出的特定格式的视觉向量中提取精确信息**。经过微调的 LLM 对"视觉 Token 里第三行第五个向量的第 78 维数值代表什么"有了专门的理解。

**变化 2：高质量指令数据**

第一代的训练数据是 GPT-4 基于"图片的文字描述"生成的——GPT-4 并没有真正看到图片。第二代的训练数据是**人工或专家标注的真实多模态问题**：

- 图表理解："比较 Q2 和 Q3 的毛利率变化，说明可能的原因"
- 文档分析："这份合同第 4 条第 2 款规定什么条件下的违约责任"
- OCR："图片左上角的蓝色小字写的是什么"
- 多图对比："图 A 和图 B 的页面布局有哪些不同"

这些问题要求模型不仅"看到"信息，还要**理解和推理**信息。

**变化 3：更高分辨率**

第一代模型通常处理 336×336 的图片——对于日常照片够了，但对于文档和图表完全不够。看一份 A4 大小的合同扫描件，336×336 下 10pt 的字迹是一团模糊。

LLaVA 1.6 提出了**动态高分辨率策略**：

```
输入：一张 1600×900 的大图
  ↓
切成多个 336×336 的子图（带重叠区域，防止切掉关键内容）
  ↓
每个子图分别用 ViT 编码
  ↓
所有子图的特征拼接起来
  ↓
送入 LLM
```

这样既保留了 ViT 的固定输入分辨率限制，又实现了对大图的细粒度理解。代价是 Token 数量翻倍——一张大图可能变成 2000+ 个 Token。

### 关键模型与突破

**LLaVA 1.5**（2023 年 10 月）

在 LLaVA 基础上做了三件事：
1. 用高质量的学术任务数据（ScienceQA、TextVQA 等）替换部分 GPT-4 生成数据
2. 将输入分辨率从 224×224 提升到 336×336
3. 换用更强的 CLIP 编码器（从 ViT-L 到 ViT-L-336px）

效果飞跃：多模态问答基准 MMBench 上从 36.2 提升到 64.3，翻了近一倍。

**LLaVA 1.6**（2024 年 1 月）

动态高分辨率策略使文档和图表理解能力大幅提升。OCRBench 得分从 210 提升到 317（提升 51%）。

**Qwen-VL**（2023-2024，阿里巴巴）

在中文场景做到了无可撼动的第一梯位。几个独有的优势：

**优势 1：原生中文训练**。LLaVA 的训练数据主要是英文，翻译到中文场景有信息损失。Qwen-VL 用中文的图文数据从头训练，对中文文档、中文路牌、中文手写文字的理解远超 LLaVA。

**优势 2：视觉定位能力**。Qwen-VL 不仅看图回答问题，还能**在图上标注位置**——你问"图里的红色按钮在哪"，它不只说"在右下角"，还会输出框的坐标 (x1,y1,x2,y2)。这对 UI 自动化、截图工具开发等场景是核心能力。

```python
# Qwen-VL 的视觉定位输出示例
{
    "question": "图中价格最高的商品在哪个位置？",
    "answer": "价格最高的商品是 iPhone，它的位置在图片右侧，标价 8999 元",
    "bbox": [450, 120, 780, 280]  # 用边界框精确标注位置
}
```

**优势 3：支持多图对话和视觉比较**。可以同时上传两张图，让模型比较差异或判断先后顺序。

**InstructBLIP**（2023 年 6 月，Salesforce）

对 BLIP-2 进行指令微调，证明了 Q-Former 架构也可以通过微调大幅提升效果，消除了"拼接式架构有天花板"的顾虑。

### 遗留问题

指令微调解决了"能力的深度"，但没有解决架构的根因——**它仍然是拼接式**。

视觉编码器和 LLM 是通过投影层"翻译"的，误差仍然存在。只是翻译得更精准了，但没有消除翻译本身。在某些复杂场景——特别是需要"一边看图一边推理，推理结果反过来影响应该看图的哪个部分"的闭环场景——第二代模型仍然力不从心。

此外，第二代的模型**仍然不能生成**——输出只有文字。GPT-4o 和 Gemini 的"统一理解和生成"是第三代的核心突破。

## 第三代：原生统一 Native Multimodal

### 思路的根本转变

```
拼接式："找个翻译"
指令微调："把翻译教得更好"
原生统一："不要翻译——让所有模态说同一种语言"
```

第三代架构不把图片和文字当作两种需要翻译的外语，而是从模型设计的第一天起，就把所有模态的数据当作**同一种 Token 序列的变体**来处理。

### 架构：没有"翻译层"

```
拼接式 / 指令微调式：
图片 → [ViT编码器] → [投影层"翻译"] → [LLM]
           ↑ 独立模块      ↑ 信息瓶颈      ↑ 另一个模块

原生统一：
图片 → [原生图片分词器] → Token → 
音频 → [原生音频分词器] → Token →  [统一的 Transformer]
文字 → [标准分词器]    → Token →
```

**关键区别**：在原生统一架构中，没有一个预先训练的 ViT 来"替你理解图片"。图片是直接被编码成一串 Token（类似 ViT 的 Patch 但更细粒度），这些 Token 和文字 Token 在**同一个自注意力计算**中被处理。

这意味着：

- **没有信息瓶颈**：原始图片的每一个像素区域都有对应的 Token，不存在 1024 个 ViT 特征向量压缩 300 万像素的情况
- **双向交互**：LLM 不只是"读到"视觉 Token——在做自注意力时，某个视觉 Token 可以"关注"一个文字 Token，文字 Token 也可以"回看"那个视觉 Token，形成迭代理解
- **生成能力**：因为所有模态共享同一个 Token 空间，模型可以输出图片 Token 而不只是文字 Token——这就是 GPT-4o 原生生图的技术基础

### GPT-4o：2024 年的范式转变

GPT-4o（"o"代表 Omni，全能）是第一个大规模商用的原生多模态模型。它在三个层面的突破定义了第三代的标杆：

**统一输入空间**

GPT-4o 的图片编码不是用 CLIP ViT 预处理，而是用一个与原模型一起训练的图片分词器。这个分词器把图片的像素区域直接映射为 Token——这些 Token 和文本 Token 共享同一个嵌入空间。

这意味着："红色物体"这个文本 Token 的向量，和图片里一块红色区域的 Token 向量，在**同一个数学空间里**。模型学会了"红色"这个概念同时出现在文本和图像两个模态中的向量对应关系。

**实时音频**

GPT-4o 对音频的处理也是一次范式升级——不再"先转文字再理解"，而是**直接理解音频**。

传统语音助手的管道：音频 → ASR（语音识别）→ 转文字 → LLM 理解 → 文字回答 → TTS（语音合成）→ 音频回答。这个管道有 3-5 秒延迟，而且情绪、语气、重音等信息在"音频转文字"这一步就丢了。

GPT-4o：音频 → 直接变成 Token → Transformer 理解 → 直接输出音频 Token → 语音回答。**没有中间的文字转换**。延迟降到 232 毫秒（GPT-3.5 语音模式的 5.4 秒缩短了 95%），而且保留了说话人的语气、节奏和情感。

**原生图片生成**

GPT-4o 可以直接生成图片 Token，不需要调用 DALL-E。这意味着——你可以在同一个对话中修改一张图的细节，而不需要每次重新生成整张图。这在之前的架构中是不可能的——DALL-E 每次调用都是从头生成。

### Gemini 2.5：视频理解的新高度

Gemini 的原生多模态架构，最大优势是**视频理解和超长上下文**。

Gemini 在做视频理解时，不是采样几帧丢给模型——它把视频当作连续的 Token 序列，用滑动窗口机制处理长视频。百万 Token 级别的上下文窗口（核心篇 07 章详细讲过上下文窗口的概念）让它可以一次性"看"完一段 1 小时的视频——不是抽 30 帧，而是几百帧甚至几千帧同时推理。

这是为什么 Gemini 的视频理解体验远超"上传视频→抽帧→GPT-4o 分析每一帧"的工作流程——原生处理连续的时间序列，能理解"这段视频从第 10 秒到第 30 秒发生了什么"，而不只是"第 5 帧、第 15 帧、第 25 帧各有什么"。

### Claude：文档与 UI 的原生多模态

Claude 的 Computer Use 是一个独特的多模态场景——不是"上传一张截图问问题"，而是**Claude 自己操作你的电脑屏幕**。它不断"看"屏幕截图作为输入，理解当前界面状态，然后输出鼠标和键盘操作。

这对视觉理解的要求比一般的看图问答高得多：它需要在复杂的 UI 界面上准确识别按钮、输入框、下拉菜单，理解它们之间的层级关系，并在操作后判断操作是否成功。

### 为什么第三代是终局方向

拼接式架构的宿命是"翻译永远会丢失信息"。指令微调想做得更好，但最好的翻译也无法还原被压缩掉的视觉细节。

原生统一消除了翻译——图片自己会说话，模型直接看。这不仅是"更好"的问题，而是"性质不同"——在拼接式架构中，模型对图片的理解永远是被动接收的二手信息；在原生统一架构中，模型有"主动看图"的能力。

但代价也是巨大的：原生统一模型的训练成本远超拼接式——需要更大规模的图文联合训练、更长的训练时间和更复杂的多模态对齐策略。这就是为什么在可预见的未来，**三代模型会共存**——不是技术上的无法替代，而是成本和应用场景的取舍。

## 三代对比与选型指南

| 维度 | 第一代 拼接式 | 第二代 指令微调 | 第三代 原生统一 |
|------|:--:|:--:|:--:|
| **代表模型** | LLaVA, BLIP-2, MiniGPT-4 | LLaVA 1.5/1.6, Qwen-VL, InstructBLIP | GPT-4o, Gemini 2.5, Claude 4 |
| **训练成本** | ~$1000 (8×A100×1天) | ~$5000-10000 | 公司级（百万美元+） |
| **本地部署** | 消费级显卡(>8GB VRAM) | 消费级到中端显卡(>16GB) | 不可能（仅 API） |
| **图片理解精度** | 粗糙，小字看不清 | 大幅提升，文档可用 | 最高，像素级理解 |
| **视频理解** | 不支持 | 抽帧勉强可用 | 原生连续时序处理 |
| **跨模态推理** | 弱 | 中等 | 强 |
| **生成能力** | 无 | 无 | 原生支持 |
| **实时语音** | 不支持 | 不支持 | 支持（232ms延迟） |
| **适用场景** | 简单图文问答、物体识别、本地原型 | 文档分析、图表解读、OCR、中文场景 | 生产级应用、复杂推理、实时交互 |

**选型指南**：

| 你的情况 | 推荐 | 原因 |
|---------|------|------|
| 个人开发者，想本地跑一个看图问答 | 第一代（LLaVA/Qwen-VL 小版本） | 消费级显卡就能跑，不需要 API 费用 |
| 企业文档处理、图表分析 | 第二代（Qwen-VL-72B/LLaVA 1.6） | 准确度够用，成本可控，可微调 |
| 中文 OCR 和文档理解 | Qwen-VL-Max（第二代） | 中文场景最强，API 价格合理 |
| 要求最高精度、复杂推理 | 第三代（GPT-4o API） | 精度第一，但成本高 |
| 实时语音交互 | 第三代（GPT-4o Realtime API） | 目前唯一可用的产品级方案 |
| 视频理解 | 第三代（Gemini API） | 原生视频处理，抽帧方案无法替代 |

## 总结

- 三代多模态架构不是简单的"越来越好"，而是**三次架构范式的重新思考**：拼接→解耦翻译→取消翻译
- **第一代解决了"能不能看图"的问题**——复用现成视觉和语言模型，训练成本低到个人能用，代价是信息压缩
- **第二代解决了"能不能理解图"的问题**——高质量指令微调 + 高分辨率输入，让模型从"看图说话"进化到"基于图推理"
- **第三代解决了"翻译即损失"的根本问题**——直接 Token 化所有模态，端到端统一理解，并原生支持生成
- **三代会长期共存**——第一代适合本地部署和原型验证，第二代是开源路线的主流选择，第三代是追求最高能力的不二之选
- 下一篇文章深入第三代架构的数学核心——看看 Transformer 到底是怎样建立不同模态 Token 之间的关联的
