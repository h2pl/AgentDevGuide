# 多模态表示与翻译

> 不同模态在模型内部怎么变成统一的数字形式？一个模态怎么"翻译"成另一个模态？这是多模态 AI 的两大底层机制。

多模态 AI 要解决两类问题。**理解侧**：文本、图片、音频这些完全不同的数据格式，模型内部怎么统一处理？**生成侧**：给一段文字，模型怎么"画出"对应的图片？给一张图片，模型怎么生成文字描述？

前者叫**表示学习（Representation Learning）**，后者叫**跨模态翻译（Cross-modal Translation）**。两件事共享"编码→隐空间→解码"的抽象逻辑，但目标不同——表示追求空间统一，翻译追求跨模态生成。

## 单模态编码：各模态怎么做数字化的第一步

每个模态的原始数据天然存在于完全不同的表示空间。模型的第一步是把它们都转成自己能计算的向量序列。

**文本编码。** Tokenizer 把句子切成 Token（词元），每个 Token 从嵌入（Embedding）表中查到一个向量。100 个词的文本 ≈ 100 个 Token，信息密度在四个模态中最高——每个向量都在承载语义。但文本有表达能力的天花板：你没法用文字精确描述一张脸上的微表情或一段音乐中的情绪转折。

**图像编码。** 主流做法是 Vision Transformer（ViT）把图片切成小块（Patch），每个 Patch 独立编码。一张 1024×1024 的图片切成 16×16 网格 = 256 个 Patch。这也是多模态 API 定价的直接逻辑——一张图 ≈ 256 个 Token。GPT-5.5 的输入价 $5/1M Token，一张 high-res 图 ≈ $0.013。缩分辨率是最有效的省钱手段，Patch 数量直接减少了。

**音频编码。** 两种主流方案：频谱图方案把波形转成梅尔频谱图，用类似图像的方式处理；Whisper 方案直接把音频波形编码成向量序列。核心挑战是序列长度膨胀——10 秒语音就能产生数千个时间帧。

**视频编码。** 负担最大的模态。1 分钟高清视频（30fps）= 1800 帧。实践中必须靠帧采样控制成本。Gemini 的默认规格是每帧 258 Token + 每秒 32 Token 音频，1 分钟视频 ≈ 18,000 Token。视频成本通常是文本的百倍量级。

## 多模态表示的两条路线

单模态编码解决"每个模态怎么变成向量"。多模态表示学习解决"不同模态的向量怎么组合"。学术界将已有方法分为两类：

**联合表示（Joint Representation）** 把多个模态投影到同一个表示空间。最简单的做法是把各模态特征向量直接拼接。GPT-5.5 和 Gemini 的架构本质上是这种思路的极致——文本 Token 和图像 Patch Token 在同一个 Transformer 中做自注意力，模型的每一层都在同时处理所有模态。优势是跨模态交互充分，劣势是所有模态必须同时输入，计算量随模态数增长。

**协调表示（Coordinated Representation）** 不把模态投影到同一空间，而是分别处理每个模态，再通过约束让它们的表示空间对齐。CLIP 是典型代表：文本编码器和图像编码器各自独立运行，用对比损失拉近匹配对、推远不匹配对。训练完成后，"猫的图片"和"a cat"在向量空间里位置相近。优势是编码器独立灵活、适合检索类任务，劣势是跨模态信息交互有限。

| 产品 | 表示方式 | 为什么选这个 |
|------|---------|------------|
| CLIP | 协调表示（对比学习） | 文本和图像编码器独立，换编码器不影响对方 |
| GPT-5.5 | 联合表示（Transformer） | 统一处理多模态，跨模态交互充分 |
| DeepSeek-V4 | 联合表示 + 专家路由 | 不同模态走不同专家路径，计算效率高 |
| Gemini 3.1 Pro | 联合表示（模态原生） | 预训练时就联合学习，无独立投影层 |

## 跨模态翻译：从一个模态生成另一个

翻译解决的是"给定一个模态的实体，生成它在另一个模态中的对应实体"。文生图、文生视频、语音合成、图生文——所有这些应用的底层都是翻译问题。

**基于示例的翻译**依赖训练数据中的配对样本，类似查字典。给定一张图片，在字典里找到最相似样本的描述直接拿来用。更先进的做法是把查询和字典都投影到共享语义空间再检索。根本局限：模型表现完全取决于字典覆盖度，遇到字典里没有的匹配时，翻译质量断崖下降。

**生成式翻译**训练一个模型从源模态直接生成目标模态，是当今主流：

- **文生图（Text-to-Image）**。DALL-E 3 用 T5-XXL 编码文本为条件向量，送入扩散模型的去噪过程。文本编码器和扩散模型是**联合训练**的——编码器主动调整输出分布配合去噪模型，因此 DALL-E 3 对 Prompt 的跟随性优于使用 CLIP 做文本编码的 Midjourney。
- **文生视频（Text-to-Video）**。Sora 2 用 DiT 架构把文本编码和视频 Patch 在统一空间中做扩散生成。视频的难度在于时序一致性——模型不仅要理解"什么物体"，还要理解"物体怎么运动"。Runway Gen-4.5 通过跨帧注意力机制保持多镜头中物体身份的连续。
- **文生音乐（Text-to-Music）**。Suno V5 用 Transformer 编码歌词和风格描述，音频解码器生成完整波形。3 分钟 44.1kHz 音频约 1600 万个采样点，歌词约 500 Token，扩展比约 1:32000。这是不同模态间信息密度差距的极端案例。
- **语音合成（Text-to-Speech）**。ElevenLabs 从文本生成语音波形，支持语音克隆和情绪控制。关键挑战是韵律生成——同样的文本在不同语境下有不同的重音、语调和节奏。
- **连续生成（Streaming Generation）**。GPT-5.5 的 Realtime API 和 Gemini Live 都支持音频流的连续生成，编码和生成几乎同时进行，需要处理流式输入和对齐时间轴。

**评估困局**。生成式翻译面临的核心问题是评估。一张图的正确描述有无数种——"一只猫站在窗台上"和"窗前有只花猫"都对。BLEU、ROUGE 等自动指标与人工判断的相关性有限。VQA 把图像理解转化为问答来部分规避编码差异，但评估问题至今未彻底解决。

## 总结

表示和翻译是多模态 AI 的两大基础能力。表示解决"怎么看"，翻译解决"怎么生成"。它们共享"编码→处理→输出"的抽象流程，但在目标、方法和评估上完全不同。理解这两个概念的区别，是后续看懂具体产品（文生图、图搜图、多模态 Agent）的前提。
