# 跨模态对齐与表示学习

编码把不同模态变成了向量，融合让这些向量产生了交互。但还有一个根本性问题：**模型怎么知道"这张猫的图"和"cat 这个词"说的是同一个东西？**

这就是**对齐（Alignment）**要解决的问题——让不同模态的表示在同一个向量空间中建立准确的对应关系。

## 为什么对齐是独立的挑战

编码和融合处理的是"形式"问题（怎么把数据变成向量、怎么让向量交互），对齐处理的是**语义**问题。

你编码一张猫的图片得到一个图像向量，编码"cat"这个词得到一个文本向量。两个向量的维度可能不同、分布可能不同、编码方式可能不同。怎么让模型知道它们"说的是同一个概念"？

这个问题不能用"把向量拼在一起"来解决——拼在一起不等于语义对齐。模型需要从海量数据中学习"什么样的图片对应什么样的文字描述"。

## CLIP：对比学习的里程碑

OpenAI 的 CLIP（Contrastive Language-Image Pre-training，2021）是对齐领域最关键的突破。它的思路简单到令人惊讶：

**收集 4 亿个"图片 + 文字描述"的配对**（从互联网上爬取的图文对），然后同时训练一个图像编码器和一个文本编码器，目标只有一个：**让配对的图文在向量空间中靠近，让不配对的互相远离。**

关键创新在于**对比损失（Contrastive Loss）**。一个 batch 有 N 个图文对，形成 N×N 的相似度矩阵。对角线是正例（配对的图文），其他都是负例。损失函数让对角线上的相似度尽可能高，让其他位置的相似度尽可能低。

这个设计的精妙之处：不需要人工标注负例，batch 内的其他样本自动成为负例。4 亿图文对 × 每 batch 数万个负例 = 海量的对齐信号。

训练完成后，CLIP 的两个编码器产生**同一个向量空间中的表示**——"猫的图片"和"cat 这个词"的向量余弦相似度很高，和"dog"的向量相似度低。

## 对齐质量决定了多模态能力的上限

如果你用开源模型做微调，对齐质量是最需要关注的指标：

**对齐好**：模型能准确理解"图片里有一只橘猫坐在沙发上"，因为"橘猫"的文字向量和图片中猫的区域向量高度对应。跨模态检索时，用"沙发上的猫"能准确搜到相关图片。

**对齐差**：模型可能把图片里的猫识别成狗，或者答非所问。更隐蔽的问题是——模型看起来能回答，但其实是靠文本侧的"猜"而不是真正的跨模态理解。对齐没做好的模型在多模态任务上的表现比纯文本 LLM 还差。

## 对齐怎么做：预训练 + 微调

### 预训练对齐

在大规模数据上学习通用的跨模态对应关系。CLIP、SigLIP 都做这一步。数据量越大，对齐越通用。

预训练对齐学到的能力：zero-shot 分类——不给任何训练样本，直接用"一张{类别名}的照片"作为 text prompt，CLIP 就能对没见过的新类别做分类。

### 指令微调对齐

在高质量"指令 + 多模态输入 + 回答"数据上微调。目的是让模型能**按照你的指令**做跨模态推理。

比如：CLIP 只能回答"这张图和这个词匹配吗"，而经过指令微调的模型能回答"图中右上角那个红色的物体是什么"。

大多数商用模型（GPT-5.5、Claude Opus 4.8）已经做好了这两步对齐。如果你在用开源模型微调，对齐应该是第一步验证的能力——用几个简单的图文匹配测试集检查对齐是否准确。

## 对齐质量怎么评估

不需要复杂的方法。实用级的评估可以用这两个测试：

1. **图文匹配**：给模型一张图片和一段描述，问"这段描述和图片匹配吗？"检查正例和反例的准确率。
2. **定位测试**：给模型一张图片和一个物体的描述，问"这个东西在图片的什么位置？"检查定位准确率。

更正式的做法参考 Arena 视觉榜和 CLIP benchmark（ImageNet zero-shot 准确率）。

## 对齐能做到什么程度

经过充分对齐的模型，嵌入空间表现出有趣的几何性质：

- **语义方向一致**。从"狗"到"小狗"的向量方向，和从"猫"到"小猫"的方向相似——说明模型学会了"幼年版本"这个语义变换
- **组合编码**。"红色的汽车"的嵌入 ≈ "红色"+"汽车"的向量组合——说明模型能理解属性组合
- **跨模态类比**。猫的图片 - cat 的文本 + dog 的文本 ≈ 狗的图片——说明文本和图像的语义空间结构一致

## 对齐的局限性

对齐不是万能的。即使最好的对齐模型也有明确的边界：

- **细节感知不足**。对颜色、位置等细粒度属性不够敏感——"左边的杯子"和"右边的杯子"可能区分不开
- **计数能力弱**。难以区分"两个苹果"和"三个苹果"——因为对比学习不强调精确计数
- **抽象概念困难**。对隐喻、幽默等高层次语义理解有限——"这张图很有趣"中包含的文化背景知识，对齐学不到
- **数据偏见**。训练数据中不均衡的分布会被编码到嵌入空间中——如果"医生"的图片大多是男性，模型可能会把"女医生"的图和"护士"关联起来

这些不是"对齐没做好"的问题，而是对比学习范式的内在限制——它学到了"什么和什么一起出现"，但没有真正理解物体的属性和关系。

## 对齐在工程中的实际影响

| 场景 | 对齐不好的表现 | 解决方案 |
|------|---------------|----------|
| 图像搜索 | "红色的车"搜出蓝色的车 | 换用 SigLIP2 等更新的对齐模型 |
| Agent 操作屏幕 | 让点击"提交按钮"却点了"取消" | 选对齐质量高的模型（Arena 视觉榜前列） |
| 多模态 RAG | 检索到不相关的图片 | 检查 CLIP 嵌入的质量，考虑 fine-tune |
| 图文一致性判断 | 图里是猫，模型说"图和文字匹配" | 换用早期融合模型（对齐+融合一起优化） |
