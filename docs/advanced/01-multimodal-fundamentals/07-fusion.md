# 跨模态融合

编码之后，每种模态都有自己的向量表示。但"猫的图像向量"和"cat 的文字向量"之间还没有直接联系——它们在各自独立的向量空间里。

**融合（Fusion）就是把这些不同来源的向量合到一起，让模型能做跨模态理解。**

不同模型采用不同的融合方式。从简单到复杂，三种策略各有优劣。

## 三种融合策略

### 晚期融合（Late Fusion）：最简单也最弱

每种模态独立处理到最后一步，只把最终特征向量拼接在一起做判断：

```
图像 → ViT → 图像特征向量 ┐
                            → 拼接 → 分类/回答
文本 → LLM → 文本特征向量 ┘
```

这是流水线方案的做法。优势是简单：可以用现成的单模态模型，不需要额外训练。劣势是模态之间只在最后一步交互——模型能说"图像识别出猫，文本提到猫，所以图文一致"，但它不知道图片的哪个区域对应文本的"猫"这个词。

**适用场景**：只需要"有没有"这种粗略判断（比如"图片和文字描述是否匹配"），不需要细粒度对应关系。

### 交叉注意力（Cross-Attention）：折中方案

每种模态保持自己的处理路径，但在某些层通过注意力机制"看"另一模态的信息：

```
文本 token 序列 ←→ Cross-Attention ←→ 图像 patch 序列
            Transformer 层          Transformer 层
```

CLIP、Flamingo、LLaVA 等模型用了这种方式。文本 token 在交叉注意力层可以"关注"到相关的图像 patch，从而知道"猫"这个词对应图片的哪个区域。

交叉注意力比晚期融合更精细，但因为模态之间需要来回传递信息，计算效率不如早期融合。

**适用场景**：需要图文对应关系的场景（"I want this chair" → 在图片中定位该椅子）。LLaVA 类开源模型多用此方案。

### 早期融合（Early Fusion）：最强也最贵

在模型最底层就把不同模态的向量拼成一个长序列，一起过 Transformer：

```
[图像patch向量1, 图像patch向量2, ..., 文本token1, 文本token2, ...]
                                    ↓
                           Transformer 自注意力
                                    ↓
                              输出向量
```

这是 GPT-5.5、Gemini 2.5 Pro 这类原生多模态模型的做法。文字 token 和图像 patch 在同一个 Transformer 中经过自注意力计算，每一层都在互相"看到"对方。

早期融合能实现最细粒度的跨模态建模——模型能编码"这个图像区域对应这段文字描述"的关系。代价是训练成本极高，需要海量多模态配对数据。

**适用场景**：需要深度跨模态推理的场景（医疗影像分析、多模态 Agent、复杂文档理解）。

## 三种策略直观对比

| 维度 | 晚期融合 | 交叉注意力 | 早期融合 |
|------|---------|-----------|----------|
| 跨模态交互深度 | 低（最后一步） | 中（部分层） | 高（所有层） |
| 训练成本 | 低（复用单模态模型） | 中（需训练对齐层） | 高（需从头训练） |
| 推理速度 | 快 | 中 | 中-快 |
| 细粒度对应 | ❌ 无 | ✅ 部分 | ✅ 完整 |
| 典型模型 | 流水线方案 | LLaVA、Qwen-VL | GPT-5.5、Gemini 2.5 Pro |

## 融合方式决定选型

- 需要深度跨模态理解（"这张图里的猫在做什么？和这段文字描述一致吗？"）→ 早期融合的原生模型
- 只需要做简单的图文匹配 → 交叉注意力或晚期融合方案更经济
- 实时场景对延迟敏感 → 早期融合（一次前向传播）优于交叉注意力（需要在模态之间来回传递信息）
- 预算有限 → 交叉注意力模型（如 LLaVA）配合高质量对齐层，在成本和效果之间取得平衡

## 对开发者的实际影响

大部分情况下你不需要关心融合策略——直接用原生多模态 API 就行。但你遇到以下情况时，理解融合策略能帮你做出更好的决策：

- **选型时**：如果场景不需要深度跨模态推理，交叉注意力方案可能够用，而且成本更低
- **调试时**：如果模型在需要细粒度对应关系的任务上出错（比如"图片左下角的数字是多少"），可能是融合策略不够深，换早期融合模型试试
- **开源部署时**：LLaVA 等交叉注意力模型是部署成本最低的多模态方案，但跨模态能力上限不如早期融合

## 总结

- 晚期融合：各管各的，最后拼结果（简单但弱）
- 交叉注意力：在中间层让模态"对话"（折中方案）
- 早期融合：从一开始就在一起处理（最强但最贵）

你的场景决定了应该选哪种。不需要深度跨模态推理时，不要为早期融合的高成本买单。
