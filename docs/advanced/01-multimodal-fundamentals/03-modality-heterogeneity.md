# 模态异质性

多模态听起来简单——不就是"支持更多输入类型"吗？但不同模态的信息天然存在于完全不同的表示空间里，这种差异叫做**模态异质性（Modality Heterogeneity）**。

把一段文字、一张图片、一段语音分别用"数字"表示出来，你会发现它们的结构完全不同，承载信息的方式也完全不同。

## 四种模态的数字表示

### 文本：离散符号序列

文本是信息密度最高的模态。每个词或子词对应一个 token，token 之间通过嵌入向量（embedding）映射到连续的向量空间。一段 100 个词的文本，变成 100 个向量组成的序列。

文本的特点：
- **离散性**。每个 token 是明确的符号，没有模糊空间。"猫"就是"猫"，不是"介于猫和狐狸之间"。
- **高语义密度**。每个 token 都承载了明确的语义。没有"冗余像素"这种概念。
- **缺乏感官信息**。你没法从文字描述中还原说话人的语气，也看不到一张图片的空间布局。

文本在信息密度上有优势，但在"表达能力"上有明确的边界——你无法用文字精确描述一张脸上的微表情。

### 图像：连续的像素矩阵

一张 224×224 的彩色图片，在计算机里是一个 224×224×3（RGB 三通道）的数值矩阵。每个像素的值在 0-255 之间，表示该位置的颜色强度。

图像和文本有两个根本区别：

**空间结构就是信息本身。** 一张猫的照片里，"猫在图片中央偏左"这种信息不是靠"描述"得到的，而是直接编码在像素的排列方式中的。你移动一个像素的位置，画面内容就变了——这在文本中不会发生（文字交换位置会产生完全不同的词）。

**信息冗余巨大。** 196 个 patch 中很多是背景，不承载关键信息。一张蓝天白云的图片，一半像素在描述"蓝色"——但你可能只关心"这是一片天空"。模型需要学会识别哪些信息重要、哪些是噪音。

### 音频：时间序列波形

音频信息以另一种方式呈现。一段 16kHz 采样率的语音，每秒产生 16000 个采样点。每个采样点是该时刻的声压值。

音频信息的特征在时间维度上展开：

- **音高**：声带振动的基频，决定了"声音听起来高还是低"
- **音色**：频谱的分布，让你区分"这是张三的声音还是李四的"
- **节奏**：音节之间的时间间隔，承载了语速和情绪信息
- **语调**：音高随时间的变化曲线，"陈述句的结尾音高下降，疑问句上升"

音频需要处理长序列。10 秒的语音 = 160000 个采样点，远超过同样时长文本的 token 数。

### 视频：空间 + 时间

视频是四种模态中最复杂的。一段 1 分钟、30fps 的视频包含 1800 帧图像，外加帧与帧之间的时序关系。

视频不仅是"很多张图片"——运动模式、事件序列、物体在时间上的连续变化，这些时序信息让视频理解的复杂度比图像高一个数量级。识别"一个人在走路"需要看到连续的步态变化，单帧图片做不到。

## 异质性带来的三个技术挑战

### 维度不匹配

一段文本的表示维度 = 几十到几百个 token。
一张图片的表示维度 = 几百到几千个 patch 向量。
一段音频的表示维度 = 数千到数万个时间帧。

模型需要统一处理这些数量级完全不同的输入。早期融合模型的做法是把它们全部拼接到同一个序列中——这意味着图像 token 会在数量上"淹没"文本 token。

### 信息密度不对等

文本 token 语义密度高但缺失感官细节。图像包含丰富的视觉信息但有大量冗余。模型需要识别：
- 哪些信息是关键的（图片中的人物、文字、关键物体）
- 哪些是冗余的（纯色背景、重复纹理、画面中的静态区域）
- 不同模态之间哪些信息是互补的（图片中的"红色" + 文字描述中的"紧急"）

### 语义鸿沟

"猫"这个文字和一张"有胡须、尖耳朵、毛茸茸的生物"的图片——两者描述的是同一个概念，但表示形式完全不同。文字是抽象符号，图像是具体的视觉特征。让模型在这两种表示之间建立准确的对应关系，是跨模态对齐的核心难题。

## 模态异质性对开发者的实际影响

| 问题 | 表现 | 怎么应对 |
|------|------|---------|
| 图像 token 太多 | 一张图抵几百个文本 token，成本飙升 | 缩小分辨率、裁剪无关区域 |
| 音频处理长 | 10 秒语音处理时间长于同等文本 | 分段处理、流式接入 |
| 跨模态定位不准 | 模型"看到了"但"说不对" | 选对齐质量高的模型（Arena 视觉榜参考） |
| 视频成本过高 | 1 分钟视频包含 1800 帧，无法全量处理 | 关键帧采样、分段摘要 |

## 模态组合成熟度

截至 2026 年中，不是所有模态组合都已成熟：

| 组合 | 成熟度 | 推荐场景 |
|------|--------|----------|
| 文本 + 图像 → 文本 | ✅ 已成熟 | 文档分析、视觉问答 |
| 语音 → 文本 | ✅ 已成熟 | 语音识别 |
| 文本 → 图像 | ✅ 已成熟 | 图像生成 |
| 文本 + 音频 → 文本 + 音频 | ⚠️ 可用 | 实时语音对话 |
| 文本 + 图像 → 动作 | ⚠️ 可用 | 屏幕控制（Computer Use） |
| 文本 + 图像 + 音频 → 综合推理 | 🔄 前沿 | 多模态 Agent |
| 实时视频流理解 | 🔄 早期 | 监控分析、直播交互 |
