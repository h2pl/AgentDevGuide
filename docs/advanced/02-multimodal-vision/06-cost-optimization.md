# 多模态视觉成本优化：从 10x 到 0.1x

多模态视觉的 token 消耗是纯文本的 10-100 倍。一张高分辨率图片的 token 消耗（~2,600 token）相当于约 2,000 个汉字。一段 1 分钟视频更是数万 token。不做成本优化，生产环境很难承受。

## Token 消耗金字塔

理解 token 消耗的分布是优化的起点：

```
一级消耗（占比 70-80%）：图像/视频帧 token
二级消耗（占比 15-25%）：历史对话上下文（多轮问答时累积）
三级消耗（占比 5-10%）：系统 Prompt + 输出 token
```

**黄金法则**：优化 image token，别纠结 Prompt 省那几个 token。

## 策略一：分辨率与细节模式

这是最有效的成本控制手段。一张图片在不同模式下的 token 差异可达 16 倍：

| 模型 | 模式 | 单张 token | 适合场景 |
|------|------|-----------|---------|
| GPT-5.5 | low res | ~255 | 缩略图、简单物体识别 |
| GPT-5.5 | high res (auto) | ~2,600 | 通用场景（默认） |
| GPT-5.5 | high res (max) | ~4,700 | 文档 OCR、小文字识别 |
| GPT-5-mini | 固定 | ~510 | 批量处理，成本压缩到极致 |
| Gemini 2.5/3.1 | 默认 | ~258 | 自动优化，一般无需手动 |
| Gemini 2.5/3.1 | 4x upscale | ~1,032 | 小文字/细节识别 |

**实践建议**：
- 默认用 low res 或 Gemini 默认值。只有发现模型看不清内容时才升级分辨率
- 对图片做预处理：裁剪无关区域、压缩到合适尺寸（1024x1024 通常足够）
- 同一任务中如果有多张图片，先分类（简单图用 low res，复杂图用 high res）

## 策略二：缓存策略

缓存在多模态场景比纯文本场景收益更大，因为每次重复调用消耗的 token 量级完全不同。

| 缓存层级 | 实现方式 | 节省比例 | 典型场景 |
|---------|---------|---------|---------|
| Prompt 缓存 | API 自带的 Prompt caching | 50%（input token 打折） | 固定 Prompt + 不同图片 |
| 图片级缓存 | 图片哈希 → 缓存结果 | 100%（完全跳过模型调用） | 重复图片（用户传同一张） |
| 分段缓存 | 视频分段结果缓存 | 60-80% | 长视频分析（分段摘要可复用） |
| Embedding 缓存 | 图片嵌入缓存 | 100%（检索阶段） | 多模态 RAG 场景 |

**Gemini Context Caching**：缓存固定 Prompt + 图片前缀，输入 token 可节省最高 67%。适合对话式场景（多轮问答中重复发送同一张图）。

**GPT-5.5 Prompt Caching**：自动缓存最近 1,000 个 token 的前缀，命中后 input 价格降半。同一 Session 中效果显著，跨 Session 无效。

## 策略三：Batch API + 模型混用

**Batch API**：所有主流模型（OpenAI、Anthropic、Google）都提供 Batch API，价格约 5 折，但延迟提升到小时级。适合非实时场景。

**模型混用策略**（实测效果最好）：

```
第一步：用便宜模型做粗筛（Gemini 2.5 Flash / GPT-5-mini）
  → 简单的直接返回结果
  → 置信度低的转入下一步
第二步：用强模型做精判（GPT-5.5 / Gemini 3.1 Pro）
  → 处理粗筛阶段不确定的样本
```

这种方案比全部用强模型节省 40-60% 成本。关键在于第一步的拒识率要设置合理——拒识率 20% 意味着 80% 的请求用便宜模型处理，仅 20% 需要升级到强模型。

## 策略四：视频成本专项优化

视频的 token 消耗远超图片，需要更激进的优化：

| 策略 | 节省比例 | 质量影响 |
|------|---------|---------|
| 降帧率（1 FPS → 0.5 FPS） | 50% | 视频内容变化慢则几乎无影响 |
| 音频降采样（32 token/s → 16） | 50%（音频部分） | 语音清晰度轻微下降 |
| 关键帧提取代替均匀采样 | 30-70% | 减少冗余帧，甚至提升质量 |
| 视频分段缓存 | 60-80%（重复分析时） | 无影响 |
| Gemini 2.5 Flash 替代 3.1 Pro | 75% | 长视频摘要场景差距小 |

**实操组合**：对于监控视频分析，用 0.5 FPS + 关键帧提取 + Gemini 2.5 Flash。相比默认方案（1 FPS + Gemini 3.1 Pro），成本降至约 1/8，质量下降通常可接受。

## 各场景成本对比

| 场景 | 默认方案 | 成本/次 | 优化方案 | 优化后成本 | 节省 |
|------|---------|--------|---------|----------|------|
| 单张图片分析 | GPT-5.5 high res | $0.013 | GPT-5-mini | $0.0006 | 95% |
| 10 页文档提取 | GPT-5.5 per page | $0.13 | GPT-5-mini + 高分辨率页提升 | $0.02 | 85% |
| 1 小时视频总结 | Gemini 3.1 Pro | $0.35 | Gemini 2.5 Flash 0.5 FPS | $0.05 | 86% |
| 100 张图片批量 | GPT-5.5 Batch API | $1.30 | GPT-5-mini Batch 8 折 | $0.05 | 96% |
| 多模态 RAG 每日 | GPT-5.5 + 无缓存 | $50/天 | 混用 + 缓存 | $8/天 | 84% |

**关键发现**：Batch API + 模型混用 + 缓存三管齐下，多数场景可降至原始成本的 10-20%。对于高吞吐场景，成本差异可能达到每月数千到数万美元。

## 成本监控与告警

建议设置三个监控指标：

1. **单次调用成本**：超过基线 3 倍时告警（说明分辨率或上下文失控）
2. **日/月累计成本**：按模型和场景分类，跟踪趋势
3. **Token 浪费率**：目标 < 10%（Prompt 中被截断或未被使用的 token 占比）
